01 — Executive Summary

Replace an 18-Year-Old System with a Platform Representatives Want to Use

Genistar's Financial Fact-Find and Game Plan tools are the engine of the advisory business — yet they run on an 18-year-old monolith that forces representatives into manual, paper-like workflows and delivers clients static PDFs they rarely revisit. The result: slower FGP preparation, lower client engagement, and a platform that actively discourages mobile-first advisors.

Kindgeek proposes rebuilding the Fact-Find and Financial Game Plan modules as a responsive, web-first application on Genistar's existing Azure microservices stack. Phase 1 delivers three high-impact capabilities: a guided, wizard-style digital Fact-Find with compliance approval workflows; an interactive Financial Game Plan dashboard that replaces static PDFs; and a simple budgeting tool for client engagement — all architected for future AI integration.

This proposal covers discovery through delivery for Phase 1, with a clear Phase 2 roadmap for the Representative Dashboard, Client Portal, gamification, and AI-powered features.

02 — Understanding the Challenge

What's Holding Genistar Back

1

Legacy Platform, Modern Expectations

An 18-year-old monolith with dated UX and no mobile responsiveness. Representatives cannot work effectively from phones or tablets — a dealbreaker for field-based advisors.

2

Excessive Manual Workflows

The Fact-Find process mirrors a paper form digitised without reimagination. Representatives spend excessive time on data entry instead of client engagement.

3

Static Outputs, Low Engagement

Clients receive Financial Game Plans as static PDFs. No interactivity, no progress tracking, no reason to return — leading to low engagement with recommendations.

4

FCA Compliance at Risk

The current system lacks robust audit trails, approval workflows, and record-keeping that FCA-regulated financial advice demands. Compliance is maintained manually.

5

No AI Readiness

Strong interest in AI-assisted workflows (meeting transcription, auto-filled Fact-Finds, AI-suggested FGPs) — but the current architecture cannot support them.

6

Infrastructure in Transition

Genistar's internal IT team is migrating to Azure microservices. The new Fact-Find/FGP module must integrate cleanly with this evolving infrastructure.

03 — Proposed Solution

Three Modules, One Platform — Built for Phase 1 Focus

Following CEO Jeff Lestz's mandate to keep Phase 1 laser-focused, we scope delivery to three core modules — no feature creep, no scope expansion beyond what directly enables representatives and clients.

Module 1: Digital Fact-Find

Transform the paper-like PDF form into a guided, wizard-style digital experience.

  • Step-by-step guided form with progress indicator (personal details, partner, dependants, employment, financial situation, existing policies, goals, risk appetite, health)
  • Save progress and resume later — representatives can complete across multiple sessions
  • UK residency criteria check (183-day rule) and referral source tracking
  • Client review and approval stage before FGP generation
  • Internal compliance approval workflow with audit trail
  • Support for multiple Fact-Finds per client (annual reviews, versioning)
  • Architecture ready for future AI meeting transcription auto-fill

Module 2: Interactive Financial Game Plan (FGP)

Replace static PDFs with a living, interactive dashboard clients actually engage with.

  • Interactive dashboard showing completed and pending activities/recommendations
  • Financial goals and progress tracking with visual indicators
  • Product recommendations display (life insurance, mortgages, investments, wills & trusts)
  • Accept/decline tracking for all recommendations
  • Calendar/timeline view for upcoming actions, payments, and review dates
  • Representative can edit/update FGP as client situation evolves
  • Version history for FCA audit compliance
  • Educational content integration for recommended products
  • External quotation system integration readiness (Vitality, etc.)

Module 3: Simple Budgeting Tool

KISS principle — a lightweight budget builder that drives client engagement.

  • Income vs. expenditure budget builder
  • Standard UK household expense categories
  • Visual budget summary showing surplus/deficit
  • Pre-population from Fact-Find financial data (no double entry)
  • API investigation for Crosslight Budget Builder integration

Cross-Cutting: Authentication, Integration & Infrastructure

  • User authentication with role-based access (Representative / Client / Compliance Admin)
  • Third-party login support (Google) — optional
  • Responsive web design — mobile-first for both representatives and clients
  • Azure microservices API integration layer
  • FCA compliance audit trail across all modules
  • GDPR compliance (consent management, data handling, UK Data Protection Act 2018)
  • New branding implementation per Genistar's rebranding guidelines

07 — Approach & Methodology

Agile Delivery with Regulatory Rigour

We follow an Agile/Scrum methodology with 2-week sprints, tailored for FCA-regulated delivery. Every sprint produces a deployable increment reviewed against compliance requirements.

1

Discovery & Requirements Validation

3–4 weeks

Deep-dive into the current Fact-Find form, FGP workflow, and compliance approval process. Technical sync with Chris Duzich and the internal IT team on Azure infrastructure and API specs. Output: validated requirements, refined wireframes, architecture blueprint.

2

UX/UI Design

4–5 weeks (overlaps with discovery)

Wizard-style Fact-Find UX, interactive FGP dashboard design, budgeting tool wireframes. Mobile-first responsive layouts. Apply new Genistar branding. Collaborative review with Aamir and stakeholders.

3

Development — Sprint Cycles

16–20 weeks

Iterative 2-week sprints. Priority order: Fact-Find module → FGP dashboard → Budgeting tool → Integration layer → Cross-cutting (auth, compliance, responsiveness). Demo at end of every sprint.

4

QA & Compliance Validation

Ongoing + 2–3 weeks final

Continuous testing throughout sprints. Final compliance validation phase covering FCA audit trail verification, GDPR data handling, security testing, and mobile responsiveness across devices.

5

UAT, Launch & Handover

2–3 weeks

User acceptance testing with Genistar representatives. Production deployment on Azure. Documentation and knowledge transfer. Transition to maintenance/support mode.

Communication & Collaboration

Daily standups (async-friendly), bi-weekly sprint reviews with stakeholders, dedicated Slack/WhatsApp channel for real-time communication, sprint retrospectives, and monthly executive status updates. Aamir Jawfer acts as Product Owner; Jeff Lestz and Chris Duzich join key milestone reviews.
